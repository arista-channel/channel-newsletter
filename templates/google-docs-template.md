# Google Docs Template for Technical Articles

This document provides instructions for creating a Google Docs template that authors can use to draft technical content for the Engineers' Exchange newsletter.

---

## How to Create the Google Docs Template

### Step 1: Create a New Google Doc

1. Go to [Google Docs](https://docs.google.com)
2. Click **"Blank document"**
3. Name it: **"Engineers' Exchange - Technical Article Template"**

### Step 2: Copy the Template Content Below

Copy everything from the "TEMPLATE STARTS HERE" section below and paste it into your Google Doc.

### Step 3: Format the Document

Apply these formatting styles in Google Docs:

**Headings:**
- Article Title (### in Markdown) → **Heading 1** (18pt, Bold)
- Section Headers (e.g., "Overview", "Technical Benefits") → **Heading 2** (16pt, Bold)
- Subsection Headers (e.g., "1. Unified Operations") → **Heading 3** (14pt, Bold)

**Text Formatting:**
- Bold terms (e.g., **Single OS**) → **Bold**
- Regular text → Normal text (11pt)
- Bullet points → Use Google Docs bullet list feature

**Colors (Optional):**
- Placeholders in [brackets] → Gray or Red to indicate they need to be filled in
- Section headers → Blue to make them stand out

### Step 4: Share as Template

1. Click **File → Make a copy** to create a template
2. Share the template with your team
3. Set permissions to "Anyone with the link can view"
4. Authors make a copy when starting a new article

## Instructions for Authors

### Before You Start

1. **Make a copy** of this template (File → Make a copy)
2. **Rename** your copy with the article title
3. **Review** the Author Guide: `templates/AUTHOR_GUIDE.md`

### Writing Tips

- **Overview**: Keep to 2-3 sentences, focus on business value
- **Key Highlights**: 3-5 bullets, use specific numbers and metrics
- **Platform Specs**: Only include if covering specific hardware/software products
- **Technical Benefits**: Group into 3-5 categories, each with 3-4 sub-benefits
- **Use Cases**: 4-6 scenarios, follow Challenge → Solution → Benefits structure
- **Partner Opportunities**: 3-5 specific business opportunities
- **Resources**: 3-5 links, mark future resources as "Coming Soon"

### Content Length Guidelines

- **Total Article**: 800-1200 words
- **Overview**: 50-75 words
- **Each Use Case**: 40-60 words
- **Each Technical Benefit**: 30-50 words per sub-benefit

### Formatting in Google Docs

- Use **Bold** for emphasis on key terms and titles
- Use bullet points for lists
- Keep formatting simple - it will be converted to Markdown

### When You're Done

1. **Review** against the quality checklist in `templates/AUTHOR_GUIDE.md`
2. **Share** the Google Doc with reviewers
3. **Convert** to Markdown and add to newsletter `index.md`
4. **Submit** PR for final review

## Converting from Google Docs to Markdown

When your article is ready, convert it to Markdown format:

### Manual Conversion

1. Copy content from Google Docs
2. Paste into a text editor
3. Add Markdown formatting:
   - `**Bold text**` for bold
   - `### Heading` for section headers
   - `- Bullet point` for lists
   - `<a href="URL" target="_blank">Link Text</a>` for external links

### Using a Converter Tool

- **Docs to Markdown** (Google Docs Add-on): Automatically converts to Markdown
- **Pandoc**: Command-line tool for document conversion
